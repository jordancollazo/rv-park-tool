# CRM Workflow: MHP/RV Acquisition

## Purpose
Manage the deal pipeline for MHP/RV park acquisitions. This directive covers the usage, maintenance, and architecture of the local CRM system built to track leads generated by `leadgen_rv_mhp.md`.

## System Architecture

### Components
1. **Database:** SQLite (`data/leads.db`) - Single source of truth.
2. **Backend:** Flask (`execution/crm_server.py`) - API and static file serving.
3. **Frontend:** HTML/JS (`static/crm.html`) - Single Page Application for interaction.
4. **Integration:** Gmail API (`execution/gmail_sync.py`) - Email tracking.

### Data Flow
`Pipeline (Scraping)` -> `CSV/DB Import` -> `leads.db` <-> `CRM UI` <-> `Gmail API`

## Usage

### 1. Starting the System
The CRM is a local web server. It must be running to view the interface.

**Quick Start (Recommended):**
```bash
# Windows
start_tool.bat

# Mac/Linux
./start_tool.sh
```

**Manual Start:**
```powershell
python execution/crm_server.py
# Access at http://localhost:8000
```

**What "the tool" means:** When the user says "start the tool", they mean the CRM map interface at http://localhost:8000 which includes:
- Interactive map with ALL leads (scraped + Crexi + LoopNet)
- 🏘️ Crexi button to toggle Crexi leads
- 🏢 LoopNet button to toggle LoopNet leads
- 📊 Pipeline button to view/manage deal stages
- Full lead details: tax shock scores, insurance scores, flood zones, opportunity metrics
- Light/dark mode toggle
- Advanced filtering and zone building tools

### 2. Importing Leads
Leads are automatically imported if you run the pipeline with the default settings (which now include DB persistence).
To manually import legacy CSVs:
```powershell
python execution/migrate_csv_to_db.py path/to/leads.csv
```

### 3. Syncing Emails
Emails are NOT real-time. You must trigger a sync.
- **Via UI:** Click "Sync Emails" button in the header.
- **Via CLI:** `python execution/gmail_sync.py`
- **First Run:** Will prompt for Google OAuth consent (uses `credentials.json` from Google Cloud Project).

## Database Schema (`data/leads.db`)

### Core Tables
- **`leads`**: high-level info (name, phone, status, score).
  - `status` enum: `not_contacted`, `contacted`, `interested`, `negotiating`, `acquired`, `dead`.
- **`calls`**: Log of phone interactions (`outcome`, `notes`).
- **`emails`**: Synced messages from Gmail.
- **`activity_log`**: Unified timeline of all events.

## Maintenance & Troubleshooting

### Missing Map Markers
- **Symptom:** Leads appear in list but not on map.
- **Cause:** Missing `latitude`/`longitude`.
- **Fix:** Run enrichment script.
```powershell
python execution/enrich_coords.py
```

### Gmail Sync Fails
- **Symptom:** Auth errors or no emails found.
- **Fix:** 
  1. Delete `token.json` to force re-authentication.
  2. Check `credentials.json` matches your Google Cloud project.
  3. Ensure user label "SENT" is accessible if syncing sent items.

### Changing Schema
If you need to add columns to the DB:
1. Manually write SQL `ALTER TABLE` statements.
2. OR: Delete `data/leads.db` (DATA LOSS WARNING) and re-run `init_db()` if in early dev.
